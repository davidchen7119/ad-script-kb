# 知识库目录结构与索引模板

## 目录结构

```
文案知识库/
├── inbox/              ← 用户丢文件的地方（唯一入口）
├── archive/            ← 处理完的分析文档按分类存放
│   ├── 政务城市/
│   ├── 企业商业/
│   ├── 文旅人文/
│   ├── 科教医疗公益/
│   └── 小众特色/
├── index.md            ← 索引导航（AI 自动维护，用户不手动编辑）
├── templates/
│   └── style_reference.md  ← 11种解说词风格模板库
└── .workbuddy/memory/  ← 工作日志（AI 自动维护）
    ├── YYYY-MM-DD.md
    └── MEMORY.md
```

## 分类 → 子文件夹映射

| 文档 frontmatter 中的 category | 目标子文件夹 |
|------|------|
| 政务城市类 | `archive/政务城市/` |
| 企业商业类 | `archive/企业商业/` |
| 文旅人文类 | `archive/文旅人文/` |
| 科教医疗公益类 | `archive/科教医疗公益/` |
| 小众特色 | `archive/小众特色/` |

## 索引文件结构

`index.md` 固定包含以下章节，新增文档时在四个表中各新增一行：

1. **风格分类体系**：5 大类 11 小类的完整说明表
2. **已入库文档**：全量文档速查表（序号 / 名称 / 分类 / 主风格 / 次风格 / 行业品牌 / 核心创意元素 / 日期）
3. **按风格检索速查**：11 种风格各自列出所有匹配文档
4. **按行业检索速查**：行业关键词 → 文档编号映射
5. **按创意元素检索速查**：创意元素 → 出现位置 → 说明
6. **风格混合使用参考**：主风格 + 有效次风格组合 → 代表案例
7. **使用说明**：AI 的工作流程指引
8. **新增文档 frontmatter 模板**：标准模板

## 索引更新规则

新增文档时，必须在 `index.md` 以下四处各新增一行：
- `## 已入库文档` 表格末尾
- `## 按风格检索速查` 对应风格小节
- `## 按行业检索速查` 对应行业行（如行业已存在则追加编号，否则新增行）
- `## 按创意元素检索速查` 中为新增的创意元素各加一行
- 如有新的主+次风格组合，更新 `## 风格混合使用参考` 表格
